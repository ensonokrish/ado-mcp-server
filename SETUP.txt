============================================================
ADO MCP Server - Setup Guide
============================================================
A generic MCP server for Azure DevOps work item management.
Works with any ADO organization and project.

Prerequisites
-------------
- Node.js 18+ installed
- VS Code with GitHub Copilot Chat

Steps
-----

1. Clone the repo
   git clone https://github.com/ensonokrish/ado-mcp-server.git
   cd ado-mcp-server

2. Install dependencies & build
   npm install
   npm approve-scripts keytar
   npm run build

3. Configure for your organization (optional)
   Edit config/defaults.json with your org-specific values:
   - organization, project, board name
   - area_path (for board scoping)
   - current_quarter (tags, iteration path)
   - product_tags (for board swimlanes)
   - required_fields (for validation)

   If config/defaults.json is removed, the server runs in generic mode
   with no enforced defaults — works with any ADO org/project.

4. MCP config is auto-generated
   After build, .vscode/mcp.json is auto-created in this repo with the
   correct path. No manual editing needed.

   To install the MCP config into another workspace:
   npm run install-mcp -- C:\path\to\your\workspace

   This auto-detects the server path based on where you cloned the repo.
   If mcp.json already exists in the target, it merges without overwriting.

5. Generate an ADO PAT
   - Go to https://dev.azure.com/<your-org>/_usersSettings/tokens
   - Create token with scopes:
     - Work Items: Read, Write, Manage
     - Project and Team: Read

6. First connection (one-time)
   Run the interactive setup wizard:

   npm run first-connect

   It will prompt you for:
     1. Organization name   - your ADO org (e.g., 'DFIN', 'Contoso')
     2. Personal Access Token - your ADO PAT
     3. Default project      - your ADO project name
     4. Profile name         - a short name to recall later (e.g., 'myorg')

   Credentials are saved to your system keychain.
   PAT never needs to be entered again.

7. Use in VS Code Chat
   - Ctrl+Shift+P -> "MCP: List Servers" -> Start ado-mcp-server
   - In chat, first connect to ADO:
     "connect with profile <your-profile>"
   - Then you can query, create, update work items:
     "show my work items"
     "create a story for..."
     "close ticket 12345"

   NOTE: You must run "connect with profile <name>" once per chat session.
   The MCP server needs to authenticate before any other tool will work.
   If you get "Not connected" errors, run connect again.

8. Disconnect
   In chat: "disconnect"
   Or from terminal: npm run disconnect

Notes
============================================================
- PAT is stored in your system keychain (Windows Credential Manager)
- No secrets are stored in files
- After first connection, use profile_name to reconnect
- Multi-project support: pass project parameter to any tool
- Multi-profile support: save different orgs/projects as profiles

============================================================
Configuration (config/defaults.json)
============================================================
All team defaults are stored in config/defaults.json:
- organization, project, board, area_path
- current_quarter (tag, iteration path)
- product_tags (board swimlane grouping)
- required_fields (validation rules)

This file is OPTIONAL. Without it, the server works generically.
With it, the server enforces your team's conventions automatically.

When a new quarter starts:
1. Update current_quarter (name, year, quarter_tag, iteration_path)
2. Commit and push — everyone gets the new defaults

============================================================
Intelligence Features
============================================================
On connect, the server loads:
- Active features/epics from your board
- Last 90 days of work items
- Team member assignment patterns

This enables:
- Auto-suggest parent feature based on title
- Auto-suggest assignee based on historical patterns
- Duplicate detection before creating items
- Spell-check on all titles (auto-corrects common typos)

============================================================
Security Posture
============================================================
Designed security-first — appropriate for regulated environments.

1. OS Keychain Credentials
   PAT stored exclusively in OS keychain (Windows Credential Manager
   or macOS Keychain). Never written to disk, env vars, or logs.
   Only the last 4 characters are ever displayed in audit logs.

2. Prompt-Injection Defence
   All tool responses are scanned by scrubToolResponse() before
   reaching the LLM. Patterns like "ignore all previous instructions",
   HTML injection, script tags, and system prompt overrides are
   detected and replaced with a safe redaction message.
   Source: src/security/scrub.ts

3. Full Audit Trail
   Every tool invocation is logged to logs/audit.log — timestamp,
   tool name, redacted arguments, success/failure, and execution time.
   10 MB x 10 rotation files. Complete traceability for compliance.
   Source: src/security/audit.ts

4. Scoped PAT Permissions
   PAT should be created with minimum required scopes:
   - Work Items: Read, Write, Manage
   - Project and Team: Read
   No repo, pipeline, or admin access needed.
